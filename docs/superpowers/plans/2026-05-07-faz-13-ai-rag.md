## Task 1: pgvector migration

```sql
-- 0013_pgvector_embeddings.sql
create extension if not exists vector;

create table public.embeddings (
  id uuid primary key default gen_random_uuid(),
  doc_type text not null check (doc_type in ('product','project','faq')),
  doc_id uuid not null,
  content text not null,
  embedding vector(1024) not null,
  created_at timestamptz not null default now(),
  unique(doc_type, doc_id)
);

create index embeddings_idx on public.embeddings using ivfflat (embedding vector_cosine_ops) with (lists = 100);
```

## Task 3: Doc serializer

Her doc tipi için "embedlenecek metin" üret:

```ts
function productToText(p: Product): string {
  return `${p.name}\n${p.short_description ?? ''}\n${p.description ?? ''}\nMarka: ${p.brand ?? ''}\nGaranti: ${p.warranty_years ?? ''} yıl\nFiyat: ${p.price} TL\nÖzellikler: ${JSON.stringify(p.specs)}`;
}
function projectToText(p: Project): string {
  return `${p.title}\n${p.description ?? ''}\nKonum: ${p.location}\nKapasite: ${p.capacityKwp} kWp\nTip: ${p.type}`;
}
function faqToText(f: Faq): string {
  return `Soru: ${f.question}\nCevap: ${f.answer}\nKategori: ${f.category}`;
}
```

## Task 4: Indexer — tüm dökümanları embed et

`lib/ai/indexer.ts`:
```ts
export async function reindexAll(): Promise<{ products: number; projects: number; faqs: number }>
```

Algoritma:
1. Tüm products/projects/faqs çek
2. Her biri için embed(text) çağır
3. `embeddings` tablosuna upsert (doc_type, doc_id unique)

Rate limiting: API rate limit'e dikkat (örn. 100/dk → her embed arası ~600ms sleep).

## Task 5: Retrieval

`lib/ai/retrieve.ts`:
```ts
export async function retrieveContext(query: string, k = 5): Promise<{ doc_type: string; doc_id: string; content: string; similarity: number }[]>
```

```sql
select doc_type, doc_id, content, 1 - (embedding <=> $1) as similarity
from embeddings
order by embedding <=> $1
limit $2;
```

## Tasarım Kararları

- **Voyage AI:** Anthropic'in önerdiği embedding sağlayıcı. Multilingual destek iyi (Türkçe için critical).
- **1024 dims:** voyage-2 default. Storage + index hızı dengesi.
- **ivfflat index:** hızlı ANN. 100 lists başlangıç için makul (DB grew → tekrar tune).
- **Top-K=5:** Anthropic context window'a sığacak, sürtünmesiz.
- **Re-index strategi:** günlük tam re-index (basit). Ürün ekleyince anlık index için Faz 14'te trigger eklenebilir.

## Bilinen Riskler

- Embedding API maliyeti: 100 ürün × günde 1 = 100 embed/gün. Voyage ücretsiz katmanı: 50M token/ay → çok rahat.
- Türkçe accent karakterler: voyage genelde iyi, ama edge case'ler test edilmeli.
- Stale data: kullanıcı ürün ekleyince hemen değil, ertesi gün re-index → 24 saat gecikme. Manuel button mevcut.
